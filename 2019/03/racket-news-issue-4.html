What’s New?

  • Stephen de Gabrielle is trying to organize a London Racket Meetup. If you are interest, get in contact with him.
  • A question by Marc Kaufmann on how to fix typos in the documentation led to the a PR by David Storrs to add a CONTRIBUTING.md to Racket. If you have some ideas on how to improve this, feel free to add a comment to the PR.
  • If you are interested in software licensing, Joel Dueck added a very interesting message to an old thread about a new BlueOakCouncil license.
  • William Bowman recently defended his PhD successfully. Congrats Dr. Bowman. On a long train ride? His thesis is available online: Compiling with dependent types.
  • Matt Jadud started a mailing list thread about pretty display of tabular data that triggered a message from Alex Harsanyi on how to reconcile all the efforts on data analysis using Racket into a single library.

If you know of library releases or maybe your own libraries and you want them to be featured, please let me know.

Project in the Spotlight

The project for this issue is Metapict by Jens Axel-Soegaard. I find this to be one of the most underrated Racket projects I know of. More people should know about it and use it!

From the docs:

The metapict library provides functions and data structures useful for generating picts. The library includes support for points, vectors, Bezier curves, and, general curves.

The algorithm used to calculate a nice curve from points and tangents is the same as the one used in MetaPost.
